## 2. 壳 → Hermes：界面语言首屏同步（`hermesdesk_lang`）

**动机**：Tauri 壳与 `http://127.0.0.1:端口` 的 Hermes 为**不同源**，`localStorage` 不共享；需在用户从壳打开控制台时，将壳当前 `getLocale()`（`zh` / `en`）传给 Hermes 首次渲染。

**做法**：

1. **`tauri/src/lib.rs`**
   - `cmd_open_hermes_dashboard` 增加参数 **`shell_locale: Option<String>`**。
   - 内部 `open_hermes_dashboard_in_webview` 在导航 URL 上追加 query：**`?hermesdesk_lang=zh`** 或 **`en`**（仅当值为 `zh` / `en` 时）。

2. **`web/src/Splash.tsx`**、**`web/src/onboarding/steps/Done.tsx`**
   - `invoke("cmd_open_hermes_dashboard", { shell_locale: getLocale() })`（`getLocale` 来自 **`web/src/lib/i18n-core.ts`**）。
   - `Done` 中 **`invoke` 失败**时的 fallback `window.location.replace` 使用：  
     `http://127.0.0.1:${port}${langQ}`，其中 `langQ` 为 `?hermesdesk_lang=...` 或空串。

3. **`hermes/web/src/i18n/context.tsx`**
   - `getInitialLocale()` **优先**读取 `URLSearchParams` 中的 **`hermesdesk_lang`**；若为 `zh` / `en`，则 **写入** `localStorage['hermes-locale']` 并作为初始语言。  
   - 否则仍用原 **`hermes-locale`** 存储，再回退到默认 `zh`。

**产品含义**：从壳进入时，控制台**首屏**与壳语言一致；用户在 Hermes 内切换语言，仍只改 **`hermes-locale`**，与壳独立（符合「两个应用叠在一起」的模型）。
