# Kabuqina 2026-04-21 改动记录

## 问题背景

Kabuqina onboarding 流程中，验证自定义 API 端点（如京东云 `https://modelservice.jdcloud.com/coding/openai/v1`）时始终报 **"Couldn't reach that API address"**。

### 根因分析

Kabuqina 架构：

1. Tauri 启动后先加载 Vite dev server（`http://localhost:5173`，此时有 Tauri IPC）
2. `lib.rs` bootstrap 用 `window.location.replace()` 把 WebView 跳转到 Python FastAPI 后端（`http://127.0.0.1:5xxxx`）
3. 跳转是设计意图——让所有网络请求走 Python 本地代理，绕过 VPN/代理限制

**代价：跳转后 Tauri IPC（`window.__TAURI_INTERNALS__`）丢失，`invoke` 变 `undefined`。**

因此：

- 原始方案（WebView2 fetch 直接请求外部 API）→ 被 WebView2 网络限制拦住
- Rust 侧 `cmd_validate_endpoint`（reqwest）→ invoke 不可用，无法调用
- 最终方案：Python 后端加 `/api/validate-endpoint` 接口，前端 fetch localhost 调用

### 2. `tauri/src/secrets.rs`

在文件末尾（`cmd_clear_secret` 之后）添加新命令：

```rust
#[tauri::command]
pub async fn cmd_validate_endpoint(
    url: String,
    api_key: String,
) -> Result<(), String> {
    log::info!("cmd_validate_endpoint called: url={}", url);
    let client = reqwest::Client::builder()
        .timeout(std::time::Duration::from_secs(10))
        .no_proxy()
        .build()
        .map_err(|e| format!("client build error: {}", e))?;
    let res = client
        .get(&url)
        .header("Authorization", format!("Bearer {}", api_key.trim()))
        .send()
        .await
        .map_err(|e| format!("Couldn't reach that API address: {} (url={})", e, url))?;
    let status = res.status();
    log::info!("cmd_validate_endpoint response: status={}", status);
    if status == reqwest::StatusCode::UNAUTHORIZED || status == reqwest::StatusCode::FORBIDDEN {
        return Err("That pass didn't work. Double-check you copied the whole thing.".into());
    }
    // 400 means the server is reachable — some OpenAI-compatible APIs return 400
    // for unauthenticated /models requests instead of 401.
    if !status.is_success() && status != reqwest::StatusCode::BAD_REQUEST {
        return Err(format!("That API address answered {}. Check the URL ends with /v1.", status));
    }
    Ok(())
}
```

### 4. `web/src/lib/validate.ts`

替换整个 `validateCustomEndpoint` 函数，实现三层 fallback 策略：

```typescript
import { findProvider, type ProviderId } from "./providers";

/** Strip trailing slashes for OpenAI-compatible base URLs. */
export function normalizeOpenAiBaseUrl(url: string): string {
  return url.trim().replace(/\/+$/, "");
}

/**
 * Check if Tauri IPC is available (i.e. we are running inside the Tauri shell).
 */
function hasTauriInvoke(): boolean {
  return typeof window !== "undefined" && !!(window as any).__TAURI_INTERNALS__;
}

/**
 * Validate a custom OpenAI-compatible endpoint.
 *
 * Strategy:
 * 1. If Tauri IPC is available → use Rust-side reqwest (cmd_validate_endpoint).
 * 2. If not (Python backend serving the page) → call a local Python API endpoint.
 * 3. If neither works → fall back to browser fetch (may fail in WebView2).
 */
export async function validateCustomEndpoint(
  baseUrl: string,
  apiKey: string
): Promise<ValidateResult> {
  const base = normalizeOpenAiBaseUrl(baseUrl);
  if (!base) {
    return { ok: false, message: "Please paste your API address (for example https://example.com/v1)." };
  }
  const trimmed = apiKey.trim();
  if (!trimmed) {
    return { ok: false, message: "Please paste your access pass." };
  }

  // Strategy 1: Tauri IPC (Rust reqwest)
  if (hasTauriInvoke()) {
    try {
      const { invoke } = await import("@tauri-apps/api/core");
      await invoke("cmd_validate_endpoint", {
        url: `${base}/models`,
        apiKey: trimmed,
      });
      return { ok: true };
    } catch (e: unknown) {
      const msg = e instanceof Error ? e.message : typeof e === "string" ? e : JSON.stringify(e);
      return { ok: false, message: msg || "Couldn't reach that API address." };
    }
  }

  // Strategy 2: Python backend proxy (when running inside Python-served page)
  try {
    const resp = await fetch("/api/validate-endpoint", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ url: `${base}/models`, api_key: trimmed }),
    });
    const data = await resp.json();
    if (data.ok) return { ok: true };
    return { ok: false, message: data.message || "Validation failed." };
  } catch {
    // Strategy 3: direct browser fetch (last resort, may fail in WebView2)
    try {
      const res = await fetch(`${base}/models`, {
        method: "GET",
        headers: { Authorization: `Bearer ${trimmed}` },
      });
      if (res.status === 401 || res.status === 403) {
        return { ok: false, message: "That pass didn't work. Double-check you copied the whole thing." };
      }
      if (res.status >= 500) {
        return { ok: false, message: `That API address answered ${res.status}. Check the URL ends with /v1.` };
      }
      return { ok: true };
    } catch {
      return { ok: false, message: "Couldn't reach that API address. Check your internet connection and the URL." };
    }
  }
}

export interface ValidateResult {
  ok: boolean;
  message?: string;
}

// validateKey 函数保持不变...
```

```python
@app.post("/api/validate-endpoint")
async def validate_custom_endpoint(request: Request):
    """Validate a custom OpenAI-compatible endpoint by hitting GET /models.

    This is called from the onboarding wizard when Tauri IPC is not available
    (i.e. the page is served by this Python backend, not by Vite dev server).
    The request body must be JSON: {"url": "...", "api_key": "..."}.
    """
    try:
        body = await request.json()
    except Exception:
        raise HTTPException(status_code=400, detail="Invalid JSON body")
    url = body.get("url", "").strip()
    api_key = body.get("api_key", "").strip()
    if not url:
        return JSONResponse({"ok": False, "message": "URL is required."})
    if not api_key:
        return JSONResponse({"ok": False, "message": "API key is required."})
    try:
        req = urllib.request.Request(url, method="GET")
        req.add_header("Authorization", f"Bearer {api_key}")
        resp = urllib.request.urlopen(req, timeout=10)
        return JSONResponse({"ok": True})
    except urllib.error.HTTPError as e:
        if e.code in (401, 403):
            return JSONResponse({"ok": False, "message": "That pass didn't work. Double-check you copied the whole thing."})
        if e.code == 400:
            return JSONResponse({"ok": True})
        return JSONResponse({"ok": False, "message": f"That API address answered {e.code}. Check the URL ends with /v1."})
    except Exception as e:
        return JSONResponse({"ok": False, "message": f"Couldn't reach that API address: {e}"})
```

## 已定稿方案：延迟跳转（优先于 Python `/api/validate-endpoint`）

**问题回顾**：整页跳到 Python 提供的 Hermes 仪表盘后，`window.__TAURI_INTERNALS__` 丢失，`invoke("cmd_validate_endpoint", …)` 不可用，自定义 Base URL 校验容易失败。

**采用做法（更简单、少暴露面）**：

1. **`tauri/src/lib.rs` — `bootstrap`**  
   - 仍照常拉起 Python 并等待 `port.txt`。  
   - 将端口写入 `AppState.hermes_port`（`Arc<Mutex<Option<u16>>>`）。  
   - **仅当** `secrets::cmd_has_secret` 为 **true** 时，才对 WebView 执行 `window.location.replace(http://127.0.0.1:{port}/)`。  
   - 若尚无密钥：**不跳转**，留在壳前端（`devUrl` 的 Vite 或打包的 `web/`），整个 onboarding 期间 **`invoke` 可用**，`cmd_validate_endpoint`（Rust `reqwest`）可正常校验京东云等自定义端点。

2. **`tauri/src/python_supervisor.rs` — `cmd_open_hermes_dashboard`**  
   - 用已保存的 `hermes_port` 拼 Hermes URL，对主窗口执行 `location.replace`。  
   - 供 onboarding **最后一页**在用户点击「Start chatting」后进入 Hermes。

3. **`web/src/onboarding/steps/Done.tsx`**  
   - 「Start chatting」改为 `invoke("cmd_open_hermes_dashboard")`；失败时再 `replace("/")` 回壳首页兜底。

**关于「延时」**：没有增加人为长等待；Python 启动耗时与改前相同。所谓「延迟」仅指 **推迟整页跳转到 Hermes 的时机**，直到已有密钥或用户主动点「Start chatting」。

**与旧草案的关系**：原先设想把壳前端构建产物复制进 `hermes/hermes_cli/web_dist` 的方案 **不再作为本问题的必做项**（且易与 Hermes 自带 SPA 混淆）。自定义 API 校验问题以 **延迟跳转 + Rust 校验** 收口。
