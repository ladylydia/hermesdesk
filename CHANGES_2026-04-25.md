## 1. 壳前端：ESLint 与 `tsconfig` 纳入 `vite.config.ts`

**背景**：在 `web/` 引入 **ESLint 9** 扁平配置（`eslint.config.js`，`typescript-eslint` 等）后，对 `vite.config.ts` 做类型感知的 lint 时，需让 **TypeScript 项目服务** 能发现该文件。

**改动**：

- **`web/tsconfig.json`**：`include` 在原有 `"src"` 基础上增加 **`"vite.config.ts"`**。

**结果**：`cd web && npm run lint` 与 `npm run build` 均可通过（以本机当时环境为准）。

## 2. Tauri：`cmd_get_hermes_port` 与 Clippy

**背景**：在 `cargo clippy -- -D warnings` 下，原先对 `Option<u16>` 使用 `clone_on_copy` 的写法已调整；随后采用 `Ok(*state.hermes_port.lock().await)` 时，在 **async 块末尾** 触发 Rust **E0597**（`State` 与 `MutexGuard` 临时量析构顺序）。

**改动**（**`tauri/src/lib.rs`**）：

- 先 **`let port = *state.hermes_port.lock().await;`** 再 **`Ok(port)`**，使锁在单条语句内释放，再返回 `Result`。

**结果**：`cd tauri && cargo clippy -- -D warnings` 通过。

### 6.1 壳前端（`web/`）

- **壳内聊天 `/chat`**：会话侧栏、与 Hermes 桥接的聊天主区、消息气泡与底部栏（含复制助手回复：图标 + 文案、占位与禁用态、时间/Hermes 模型名）。
- **品牌展示**：`BrandMark` 使用 `public/logo.png`（加粗 H 的定稿），顶栏/引导/启动页等位置与 **favicon** 对齐；`index.html` 中 PNG/SVG 图标关系已理顺。
- **设置页**：区分 **普通用户** 与 **高级用户** 下「工作区」区块——普通用户强调 **沙箱工作区**、不展示路径与「打开文件夹」；高级用户仅给 **短说明 + 路径 + 打开**。**高级用户**说明中注明：开/关后会 **自动重启本机助手进程** 以使工具集生效（与 6.3 一致)。
- **构建脚本**：`package.json` 的 `build` 从 `tsc -b` 改为 **`tsc --noEmit`**，避免在 Windows 上写 `tsconfig.tsbuildinfo` 时出现 `UNKNOWN: open`；根目录 **`.gitignore`** 增加忽略该 buildinfo 文件（若再生成也不会误交）。

### 6.2 Tauri（`tauri/`）

- **高级用户开关与真实权限一致**：`cmd_set_power_user` 在写入设置后 **重启嵌入式 Python 子进程**，使子进程内 **`HERMESDESK_POWER_USER`** 与 `python/overlays/default_toolset.py` 的 seed 与 UI 一致（此前仅改设置、不重启，会出现「关高级仍能用终端」的错位）。
- **复用环回桥参数**：`AppState` 增加缓存 **bridge 的 secret/approval URL**，复连同一 bridge，不必二次 `bridge::spawn`。
- `paths` 中保留 **`set_power_user_enabled`** 供 `lib` 在重启前写盘；原 `cmd_set_power_user` 由 `lib` 里异步命令实现。
- **品牌资源**：以根目录 `logo.png` 跑 `cargo tauri icon` 生成/更新 `tauri/icons/*`、`icon.ico`、**`tray.png`（自 32×32 复制）** 等（若 Android 某文件被占用，可能仅报该步；**Windows 桌面/安装包**所需项仍会生成）。

### 6.3 嵌入式 Python（`python/`）

- **`overlays/desk_system_prompt.py`**：在 `desktop_entrypoint` 在 **`import hermes_cli.web_server` 之后** 安装对 **`run_agent.AIAgent._build_system_prompt`** 的包装。  
  - 当 **`valid_tool_names` 中不含** `terminal`、`browser_*`、`execute_code`、`mixture_of_agents` 时，在系统提示中说明 **未开高级能力**，并在用户需终端/本机浏览器/沙箱代码等时 **明确引导** 到壳内 **设置 → 高级用户模式**（含中英文名），避免模型假装有工具。  
  - 若已具备上述能力，则提示高级模式为开、高风险步骤仍以真实 tool 结果为准。  
- **开发目录**：若 `overlays` 在 `python/` 下、入口在 `python/src/`，对 `desk_system_prompt` 的 import 增加了 **`sys.path` 回退**，与 `apply_all` 行为一致。
