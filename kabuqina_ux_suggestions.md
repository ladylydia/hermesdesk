# Kabuqina / 卡布奇娜 UX 改进建议整理

> 产品背景：Kabuqina 是一个面向 non-tech 用户的桌面 AI agent，目前主要是 C 端窗口形态，后续可以逐步演进为更有“桌面存在感”的个人 AI 助手。

---

## 1. 产品定位建议

Kabuqina 不一定要和 Hermes、Claude Desktop、OpenAI Operator 这类产品直接拼“最强 agent 能力”。

更适合的方向是：

> 做一个让普通用户愿意长期打开、长期挂在桌面上的个人 AI 助手。

核心差异不是“能不能自动化”，而是：

- 用户是否知道怎么用
- 用户是否觉得安心
- 用户是否愿意每天打开
- 用户是否觉得她像一个助手，而不是一个复杂工具
- 用户是否能形成轻度依赖

一句话定位可以是：

> Kabuqina 是一个为普通用户设计的桌面 AI companion，帮你处理电脑上的小事，陪你轻松工作。

---

## 2. 现在窗口形态的整体判断

当前 UI 已经有一些比较好的基础：

- 干净
- 不压迫
- 有 desktop app 感
- 没有明显的“AI 控制台”焦虑
- 顶部导航简单
- 输入区不复杂
- 大留白让产品看起来比较轻

但是目前整体仍然更像：

> 一个聊天软件 / ChatGPT-like app

还没有完全进入：

> 桌面助手 / companion product

下一阶段不一定要大幅增加功能，而是应该重点提升两个区域：

1. 左侧栏
2. 中间空状态区域

---

## 3. 左侧栏：从“聊天历史”变成“助手工作记录”

### 当前问题

现在左侧栏主要是历史聊天列表，这会传达出一种感觉：

> 这是一个 chat app。

而不是：

> 这是一个会帮我做事情的 assistant。

对 non-tech 用户来说，他们未必关心“我之前开过哪些聊天”，他们更关心：

- 小娜帮我做过什么
- 哪些提醒还在
- 哪些任务完成了
- 我能不能快速回到某个任务

### 建议方向 A：任务 / 记录型左栏

可以把左栏从“聊天历史”逐步调整为“助手工作记录”。

例如：

```text
今天
☕ 提醒我喝水
🎵 提醒我唱歌
😴 提醒我睡觉
📎 分析了 1 张图片

最近
🛠 帮我安装/配置工具
📄 读取了一个文件路径
💬 关于小娜的自我介绍
```

这样会更像：

> 小娜最近帮你做过什么。

而不是：

> 你以前聊过什么。

### 建议方向 B：保留聊天，但弱化 ChatGPT 感

如果当前阶段不想重构左栏，也可以做轻量优化：

- 把重复标题合并或自动改写
- 把“你是谁？”这类重复会话整理成更自然的摘要
- 按“今天 / 昨天 / 最近”分组
- 给提醒类任务加 icon
- 给文件类任务加 icon
- 给图片类任务加 icon

例如把：

```text
你是谁？
你是谁？
你是谁？
1 分钟后提醒我喝水
1 分钟后提醒我唱歌
```

改成：

```text
今天
☕ 喝水提醒
🎵 唱歌提醒
💬 小娜的自我介绍
```

### 左侧栏的目标

左侧栏不应该只是历史记录，而应该承担这几个角色：

- 用户知道小娜最近做过什么
- 用户可以快速回到任务
- 用户能看到提醒、文件、图片等任务类型
- 用户感觉小娜是“助手”，不是“聊天窗口”

---

## 4. 中间空状态：从“品牌展示”变成“行动入口”

### 当前问题

中间区域的 Logo 和 slogan 是干净的，但现在有一个明显问题：

> 用户看完以后，不知道下一步要干嘛。

对技术用户来说，他们可能会主动输入需求；但 non-tech 用户经常不知道：

- 该问什么
- 小娜能做什么
- 怎样表达任务
- 什么任务适合交给 AI

所以中间空状态需要承担“轻引导”的作用。

### 建议加入轻量 suggestion chips

可以在 Logo 和 slogan 下方加入 4 到 6 个可点击建议动作。

例如：

```text
帮我设置一个提醒
帮我整理下载文件
帮我总结图片内容
帮我找刚刚打开的文件
帮我压缩几张图片
帮我写一段消息
```

这些建议应该避免技术语言，不要写成：

```text
Run workflow
Execute tool
Use agent mode
Analyze context
```

更适合写成普通用户能直接理解的话：

```text
提醒我 30 分钟后休息
帮我看看这张图片
帮我整理桌面文件
帮我把这段话写得更自然
```

### 建议结构

中间区域可以保持简洁：

```text
Kabuqina

How about a Nana special?

[提醒我一件事] [整理文件] [总结图片] [写一段消息]
```

或者中文版本：

```text
卡布奇娜

今天想让小娜帮你做点什么？

[设置提醒] [整理文件] [总结图片] [写消息]
```

### 中间区域的目标

中间空状态不只是装饰，而应该变成：

- 新用户的第一步引导
- 功能发现入口
- 降低表达成本
- 让用户知道小娜不是只能聊天，而是能帮忙做事

## 5. 悬浮态 / 缩小版桌面 UX 建议

当前 Kabuqina 还没有悬浮在 desktop 的模式，也没有主动发声模式。这个选择是合理的，不建议一开始就做过重的桌面宠物化。

更适合的演进路线是：

> 窗口助手 → 轻悬浮状态 → 上下文建议 → 桌面 companion

### 悬浮态的定位

悬浮态不要一开始承担完整聊天功能。

更适合定位为：

> 低干扰的桌面存在感。

也就是：

- 用户知道小娜在
- 但小娜不打扰用户
- 需要的时候可以快速唤起
- 有任务和提醒时可以轻轻出现

### 悬浮态可以承担的功能

适合放在悬浮态里的内容：

- 当前状态
- 任务进度
- 定时提醒
- 轻量建议
- 一键展开主窗口
- 一键稍后提醒
- 一键完成 / 忽略

不适合一开始放在悬浮态里的内容：

- 完整聊天窗口
- 长文本回复
- 复杂设置
- 多步骤 workflow
- 大面积弹窗

### 状态示例

```text
☕ 待机
✨ 正在处理
⏰ 提醒到了
📎 发现可整理内容
😴 安静模式
```

### 提醒 UX 示例

普通提醒软件可能是：

```text
Drink water.
```

小娜可以更像 companion：

```text
该喝点水啦 ☕
```

或者：

```text
你已经工作一会儿了，要不要休息一下？
```

但要注意：语气要克制，不要太“卖萌”，否则容易显得幼稚或打扰。

---

## 6. “低干扰”是核心原则

面向 non-tech 用户时，Kabuqina 不应该给人一种“AI 正在接管电脑”的感觉。

尤其要避免：

- 突然弹大窗口
- 自动播放语音
- 自动执行很多操作
- 长时间显示复杂执行日志
- 一直主动冒出来
- 让用户觉得被监视

更适合的原则是：

> 小娜可以主动，但一定要轻。

### 好的主动方式

```text
这些文件看起来有点乱，要不要我帮你整理一下？
```

### 不好的主动方式

```text
检测到文件系统存在可分类对象，是否执行自动归档 workflow？
```

前者像助手，后者像企业软件。

---

## 7. 失败体验需要特别设计

Agent 产品最大的问题不是成功，而是失败时很尴尬。

普通用户遇到失败时，不会像技术用户一样理解 tool error、权限问题、模型误判。他们会直接觉得：

> 这个 AI 不靠谱。

所以失败体验要尽量人性化。

### 不建议

```text
Tool execution failed.
```

```text
Error: permission denied.
```

```text
Workflow aborted.
```

### 更建议

```text
这个步骤我没成功，我换个方式试试。
```

```text
我现在没有权限处理这个文件，你可以先把文件拖进来。
```

```text
这次我可能理解错了，你想让我处理哪一个文件？
```

失败体验的目标不是掩盖错误，而是：

- 让用户知道发生了什么
- 不让用户紧张
- 给出下一步
- 保持对小娜的信任感

---

## 8. Reminder 是非常适合 Kabuqina 的早期特色功能

提醒功能天然适合 companion 产品。

原因：

- 高频
- 低风险
- 容易形成习惯
- 不需要复杂权限
- 适合悬浮 UX
- 能体现“小娜在陪你”

可以优先强化这些提醒场景：

- 喝水提醒
- 休息提醒
- 睡觉提醒
- 会议前提醒
- 文件发送提醒
- 下载完成提醒
- 工作节奏提醒

提醒体验要避免太像普通闹钟，而是更像轻轻提醒：

```text
该喝点水啦 ☕
```

```text
会议还有 10 分钟，要不要我帮你准备一下？
```

```text
已经很晚啦，要不要收个尾？
```

## 9. 后续功能方向：更适合 non-tech 用户的特色

相比 Hermes 偏技术 / agent framework 的能力，Kabuqina 可以更偏普通用户的电脑生活场景。

### 适合优先探索的方向

#### 1. 电脑生活整理

普通用户真正痛苦的事情包括：

- 下载文件乱
- 桌面文件乱
- 截图找不到
- PDF 不知道怎么处理
- 图片不会压缩
- 文件命名混乱
- 微信 / 邮件附件到处都是

小娜可以做：

- 自动归类下载
- 帮截图命名
- 提醒重复文件
- 压缩图片
- 一键打包文件
- 找“昨天那个 PDF”

#### 2. 自然语言操作电脑

普通用户不会说 workflow，但会说：

```text
帮我把今天下载的图片缩小一点
```

```text
帮我找刚刚那个 PDF
```

```text
提醒我 20 分钟后继续这个任务
```

这类表达很适合 Kabuqina。

#### 3. 轻量观察式帮助

后续可以谨慎做“观察式帮助”，但要非常注意隐私感和打扰感。

例如：

```text
你刚下载了好几个文件，要不要我帮你整理一下？
```

不要做成：

```text
我检测到你正在进行文件管理行为。
```

## 10. UI 设计原则总结

Kabuqina 的 UI 可以围绕这些关键词：

- 安静
- 轻
- 圆润
- 低压
- 不打扰
- 有陪伴感
- 不像企业软件
- 不像开发者工具
- 不需要学习
- 随时可打断

每个功能上线前可以问几个问题：

- 这个功能会不会让用户紧张？
- 会不会太像自动化系统？
- 会不会突然打扰用户？
- 用户是否需要学习概念？
- 用户是否知道下一步该做什么？
- 失败时是否足够温和？
- 这个功能是否更像“小娜”，而不是 Hermes？

## 11. 推荐的版本演进路线

### 当前版本

窗口型 AI assistant。

目标：

- 保持干净
- 完成核心对话和任务能力
- 不急着做复杂桌面化

### 下一版本

重点优化：

- 中间空状态加入 suggestion chips
- 左侧栏从聊天历史改向任务记录
- 提醒功能更有 companion 感
- 优化重复历史标题
- 强化 non-tech 用户的第一步引导

### 再下一版本

探索：

- 轻悬浮态
- 提醒悬浮 UX
- 任务进度悬浮状态
- 一键展开主窗口
- 安静模式

### 更后续版本

谨慎探索：

- 上下文感知建议
- 文件整理助手
- 桌面 companion 化
- 更细的动画和状态人格

---

## 12. 最重要的一句话

Kabuqina 不需要成为最强的 agent。

她更应该成为：

> 最舒服、最容易被普通用户长期使用的桌面 AI 助手。

也就是说，Kabuqina 的护城河可能不是：

- tool 数量
- agent loop
- workflow 复杂度
- benchmark 能力

而是：

- UI 情绪
- 陪伴感
- 低干扰
- 用户安全感
- 任务表达门槛低
- 失败体验温和
- 普通用户真的愿意每天打开

这才是 Kabuqina 和 Hermes 最应该拉开的差异。
