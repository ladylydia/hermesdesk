## 2. Gateway API key 桥接（secret_loader）

### 2.1 `hermes/gateway/run.py` — `start_gateway()`

- 在 `start_gateway()` 入口处注入 `secret_loader.install()` 调用
- 原理：Tauri 壳将用户 API key 存在 Windows Credential Manager 中，通过一次性 loopback URL（`HERMESDESK_SECRET_URL`）分发给子进程
- **此前**：仅 `desktop_entrypoint.py` 启动的 Hermes web_server 进程调用了 `secret_loader`，Gateway 进程虽收到 `HERMESDESK_SECRET_URL` 但从未 fetch，导致微信/QQ bot 接收消息后无 API key 调用 LLM
- **此后**：Gateway 启动时自动从 Tauri 桥获取 `OPENAI_API_KEY` / `HERMES_INFERENCE_PROVIDER` / `OPENAI_BASE_URL`，微信与 QQ bot 均可用

### 2.2 `python/overlays/secret_loader.py`（参考）

- `install()` 读取 `HERMESDESK_SECRET_URL` → loopback HTTP fetch → 写入 `OPENAI_API_KEY`（custom provider）或对应 env

## 3. QQ Bot 桌面扫码配置（Route C）

### 3.1 `python/src/qqbot_qr_worker.py`（**新建**）

- 镜像 `weixin_qr_worker.py`，使用 QQ Bot 扫码 API：
  1. `create_bind_task()` → 获取 `task_id` + AES key
  2. `build_connect_url(task_id)` → 生成 QQ 扫码 URL

## 4. Windows WinError 87 修复（两处）

### 根源

Python `os.kill(pid, 0)` 在 Windows 上对无效/已退出 PID 抛出 **`OSError` (WinError 87: 参数错误)**，而非 Linux 上的 `ProcessLookupError`。

### 4.1 `hermes/gateway/status.py:579` — `get_running_pid()`

- `except (ProcessLookupError, PermissionError)` → `except (ProcessLookupError, PermissionError, OSError)`
- **症状**：上次 gateway 运行留下的 `gateway.pid` 中含旧 PID，新进程检测时 `os.kill(pid, 0)` 崩出 `SystemError`
- **影响**：Gateway 启动即 crash（exit code 1）

### 4.2 `hermes/gateway/status.py:344` — `acquire_scoped_lock()`

- 同样补 catch `OSError`
- **症状**：QQ Bot 连接时 `_acquire_platform_lock("qqbot-appid", ...)` 读到旧 lock 文件中的 stale PID，`os.kill(pid, 0)` 崩出 `OSError`
- **影响**：QQ Bot 始终无法连接，日志 `✗ qqbot error: [WinError 87]`，进入重连死循环

### 清理

- 删除 `%USERPROFILE%\.local\state\hermes\gateway-locks\` 下残留的 `qqbot-appid-*.lock` 和 `weixin-bot-token-*.lock`
