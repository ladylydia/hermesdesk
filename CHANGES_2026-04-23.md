# Kabuqina 2026-04-23 改动记录

本文件汇总 **2026-04-23** 日前后在仓库中落地的功能与修复（含壳 `web/`、Tauri `tauri/`、Hermes 子模块 `hermes/`）。与 `CHANGES_2026-04-21.md`、`CHANGES_2026-04-22.md` 为同一套记录风格，可对照阅读。

## 1. 启动与导航：不再自动跳进控制台

**问题**：已保存凭据时，`bootstrap` 在 Python 端口就绪后立刻 `navigate` 到 `http://127.0.0.1:端口/`，用户未经壳内说明就直接进入 Hermes 页。

**做法**：

- `tauri/src/lib.rs`：**删除**「有 `cmd_has_secret` 则自动 `open_hermes_dashboard_in_webview`」的逻辑；启动后始终先停留在 **Tauri 壳**（`devUrl` / 打包的 `web/dist`）。
- 用户从 **Splash 的「打开控制台」** 或 **onboarding 完成页的「进入控制台」**（英文化界面为 *Start chatting*）再 `invoke("cmd_open_hermes_dashboard", …)` 进入 Hermes（见下文 **语言同步**）。
- 使用 `url::Url` + `WebviewWindow::navigate`（**不要**使用私有的 `tauri::webview::WebviewUrl` 导入；`reqwest` 使用 `native-tls` 等，见 `tauri/Cargo.toml`）。

## 3. 壳前端：中文优先 + 可切换英文（i18n）

- 新增 `web/src/locales/strings.ts`（`MESSAGES.zh` / `MESSAGES.en`）。  
- `web/src/lib/i18n-core.ts`：`translate`、`getLocale` / `setStoredLocale`、`localStorage` 键 `hermesdesk.locale`、**默认 `zh`**。  
- `web/src/lib/i18n.tsx`：`I18nProvider`、`useI18n()`、`useLayoutEffect` 同步 `document.documentElement.lang`（`zh-CN` / `en`）。  
- `web/src/components/LanguageToggle.tsx`：**中文 | English**。  
- `web/src/main.tsx` 根上包裹 `I18nProvider`；`web/index.html` 默认 `lang="zh-CN"`。  
- 已接文案的页面/模块：Splash、ShellFrame、onboarding 各步、Settings、`validate.ts` 用户可见错误提示等。

## 4. 密钥与 `settings.json` 一致性

- `tauri/src/secrets.rs`： `**cmd_clear_secret` 时删除 `settings.json` 中的 `provider` 段**（`clear_provider_cfg`），避免「keyring 已清、配置残留」导致状态不一致。

## 5. 自定义 API 校验与构建脚本

- `**web/src/lib/validate.ts`**：`validateCustomEndpoint` 为 **Tauri `invoke` → `cmd_validate_endpoint` →（失败时）POST `/api/validate-endpoint` → 再浏览器 `fetch`；`invoke` 使用静态 `import("@tauri-apps/api/core")` 的替代方案为文件顶部静态 `import`，避免 Vite 对同一模块既动态又静态引用的告警。  
- `**hermes/hermes_cli/web_server.py`**：包含 `**POST /api/validate-endpoint`**（`urllib` 探测），路径在 `**_PUBLIC_API_PATHS**` 中。  
- `**python/build_bundle.ps1**`：`pip install` 增加  
`--platform win_amd64 --python-version 3.11 --only-binary=:all:`（缓解 standalone Python 与 wheel 问题，见 04-21 记录中的待办）。

## 6. Hermes 控制台（`hermes/web`）：默认中文 + 与壳语言同步

- `hermes/web/src/i18n/context.tsx`：无 `localStorage['hermes-locale']` 时默认 `**zh**`（原为 `en`）；占位 `Context` 默认 `zh` + `t: zh`；`useLayoutEffect` 设置 `<html lang>`。  
- `hermes/web/index.html`：`lang="zh-CN"`。  
- **说明**：若用户此前从未写入语言，首次打开控制台将为中文；**已**存 `en` / `zh` 的仍尊重用户选择。  
- **从壳进入时**：若 WebView 导航的 URL 带查询参数 `hermesdesk_lang=zh|en`（与壳 `getLocale()` 一致），Hermes 首次加载会采用该语言并**写入** `hermes-locale`（与壳的 `hermesdesk.locale` 为不同源下的两套存储，通过 URL 对齐偏好）。

## 7. Hermes 内：桌面试聊原型（Beta / 试用）

**目标**：在**不先恢复独立 gateway 进程**的前提下，打通 **浏览器 + session 令牌 + 本地服务** 的最小闭环，为后续接 agent 留接口。


| 项    | 说明                                                                                                                                                                                     |
| ---- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 顶栏   | 在 **Status** 之后增加路径 `**/desk-chat`**，**第二个 tab**；`Bot` 图标；`beta: true` 时显示 **试用** / **Beta** 徽标。                                                                                       |
| 页面   | `hermes/web/src/pages/DeskChatPage.tsx`：说明文案 + 输入 + 发送 + 结果区。                                                                                                                          |
| API  | `POST /api/desk/chat-proto`（`web_server.py`），**需 session token**（**不在** `_PUBLIC_API_PATHS`）。请求体 `{"message": "..."}`，成功返回 `ok`、`proto`、`received_chars`、`preview`（**不接完整 agent/网关**）。 |
| 前端   | `hermes/web/src/lib/api.ts` 增加 `deskChatProto` 与类型 `DeskChatProtoResponse`。                                                                                                            |
| i18n | `hermes/web/src/i18n/types.ts` + `zh.ts` / `en.ts`：`app.nav.deskChat`、`deskChatPage` 下各键。                                                                                              |
