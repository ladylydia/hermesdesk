## 1. 消息网关（Tauri + 壳设置页）

### 1.1 `tauri/src/lib.rs`

- **`cmd_gateway_start`**：启动后等待 **2s** 再 `try_reap`；若子进程已退出，错误信息拼接 **`gateway_state.json` 的 exit_reason**、`hermes-home/logs/gateway.log` **尾部**，并提示执行 **`python/build_bundle.ps1`**。
- 若检测到嵌入式 **`hermes/gateway/run.py` 不含首轮保活补丁**（见 `gateway_supervisor::bundled_gateway_has_startup_survival`），在错误末尾追加**明确英文说明**（避免仅看到泛化提示仍不知道包已过期）。
- **`cmd_gateway_status`**：返回 **`embeddedGatewayStartupSurvival`**（布尔），供设置页展示「嵌入包 gateway 是否为新逻辑」。
- **`ensure_gateway_after_hermes_respawn`**（嵌入式 Hermes 重启后）：在 `.env` 具备消息凭据时尝试拉起网关（与手动启动路径一致）。

### 1.2 `tauri/src/gateway_supervisor.rs`

- **`parse_dotenv_upper`**：读取 `.env` 时去掉 **UTF-8 BOM**，避免首行键名带 `\uFEFF` 导致读不到 `WEIXIN_*`。
- **`WeixinEnvSnapshot`**：增加 **`has_account_id` / `has_token`**（仍不返回密钥）；**`read_weixin_env_snapshot`** 供 `cmd_weixin_env_status` 使用。
- **`bundled_gateway_has_startup_survival`**：读取 **`{bundle_dir}/hermes/gateway/run.py`** 文本，检查是否同时包含 **`keep the process alive`** 与 **`_platform_reconnect_watcher`** 片段，用于判断 bundle 是否包含「首轮全失败仍可保活」的上游修复。

### 1.3 `web/src/advanced/Settings.tsx` + `web/src/locales/strings.ts`

- **启动网关**：`gatewayStarting` 状态、**单飞 ref**（避免连点并发多次 `invoke`）、启动中禁用「停止网关」；按钮与状态行文案 **`gatewayStarting` / `gatewayStatusChecking` / `gatewayStartingHint`**。
- **嵌入包偏旧**：当 **`embeddedGatewayStartupSurvival !== true`** 且已有消息凭据时，展示 **`gatewayEmbedStale`** 琥珀色说明（强调 Keys 已配微信仍可能 **exit code 1**，需重新 **`build_bundle`**）。
- **`GatewayStatusResponse`** 类型扩展 **`embeddedGatewayStartupSurvival`**。

## 3. Hermes 子模块（`hermes/`）

> 以下变更在 **git 子模块** 内；**桌面端实际运行的是 `python/dist/runtime` 里的拷贝**，改完子模块后必须 **`.\python\build_bundle.ps1`** 再启动 HermesDesk。

- **`hermes/gateway/run.py`**：当所有已启用平台**首轮连接失败**但错误归类为**可重试**时，**不再**在 `start()` 阶段直接让进程退出；写入 **`gateway_state`** 为可恢复状态并继续跑 **`_platform_reconnect_watcher`**（缓解微信/iLink 冷启动与 HermesDesk「秒退」观感）。
- **`hermes/hermes_cli/config.py`**：将 **`WEIXIN_*`** 纳入 **`OPTIONAL_ENV_VARS`**（与 Keys `/api/env` 计数、展示一致）；避免微信变量只出现在 `_EXTRA_ENV_KEYS` 导致仪表盘「消息类」统计偏差。
- **`hermes/hermes_cli/gateway.py`**：**`_setup_weixin`** 在 QR 成功后与 **`weixin_qr_worker`** 相同的 **`remove_env_value` + 再写入** 逻辑（§2.4）。
- **测试**：与上述网关启动行为相关的单测期望已调整（若子模块内有对应文件）。

## 6. 使用与排错（摘要）

1. **拉取或修改 `hermes/` 后**：关闭 HermesDesk → **`.\python\build_bundle.ps1`** → 再 **`cargo tauri dev` / 安装包**。否则嵌入式 **`hermes/gateway/run.py`** 可能仍旧，首轮微信失败会 **exit 1**。
2. **设置 → 消息网关** 若出现 **「嵌入包 gateway 偏旧」**：同上，先重建 **`python/dist/runtime`**。
3. **个人微信「已配置」**：以 **`应用数据目录/hermes-home/.env`** 中 **`WEIXIN_ACCOUNT_ID` + `WEIXIN_TOKEN` 均非空**为准；与网关是否连上无关。仅缺一显示黄色「凭据不完整」条。
4. **QQ 机器人**：在 Keys 保存 **`QQ_APP_ID`** 与 **`QQ_CLIENT_SECRET`** 后，**消息网关**应变为可启动；可用 QQ 先于个微验证网关是否正常（个微 iLink 若不稳定不影响 QQ 路径）。
5. **飞书**：在 Keys 保存 **`FEISHU_APP_ID`** 与 **`FEISHU_APP_SECRET`** 后同上；设置页有 **`FeishuSettingsBlock`** + **`cmd_feishu_env_status`**（见 `tauri/src/feishu_env.rs`、`gateway_supervisor::FeishuEnvSnapshot`）。
6. **Telegram**：**`TelegramSettingsBlock`** + **`cmd_telegram_env_status`**；**`TelegramEnvSnapshot`**（`TELEGRAM_BOT_TOKEN` 脱敏预览；仅有其它 `TELEGRAM_*` 而无 token 时琥珀提示）。
7. **微信 .env 冲突 / 重复行**：扫码保存前 **`remove_env_value`** 清理 `WEIXIN_APP_ID`、`WEIXIN_APP_SECRET`（教程遗留）及对 **`WEIXIN_ACCOUNT_ID` / `WEIXIN_TOKEN` 的全量行**再写入，避免 `save_env_value` 只改首行留下旧副本（见 **`docs/troubleshooting.md` §13**）。**`python/src/weixin_qr_worker.py`** 与 **`hermes/hermes_cli/gateway.py`** 的 **`_setup_weixin`** 同步。
