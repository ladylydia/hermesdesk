# Kabuqina 2026-04-22 改动记录

## 问题背景

Kabuqina 启动后直接跳转到 Hermes Python 后端的 dashboard，而不是进入 onboarding 向导。

Dashboard 显示：
- 代理 v0.10.0 在线
- 网关未运行（离线）
- 活跃会话无

尝试在 dashboard 创建会话失败。

### 根因分析

1. **onboarding 跳过问题**：`cmd_has_secret` 返回 true，因为：
   - `read_provider_cfg` 从 settings.json 读取到 provider 配置
   - 即使 keyring 里的 secret 已删除，settings.json 可能还有残留

2. **dashboard 无法聊天**：Kabuqina 架构设计问题
   - Hermes 原本有四个入口：CLI、TUI、Gateway、Web UI
   - Kabuqina 剥离了 CLI、TUI、Gateway（见 `python/overlays/strip_shims.py`）
   - 只剩下 Web 管理后台（Status/Sessions/Config 等页面），这不是聊天界面

## 已完成的临时修复

### 1. `tauri/src/lib.rs`

强制不跳转到 Python 后端，保持 onboarding UI：

```rust
if let Some(w) = app.get_webview_window("main") {
    // Force onboarding for now — always show the shell UI
    log::info!("forcing onboarding UI");
    let _ = w.show();
    let _ = w.set_focus();
}
```

添加 `cmd_get_hermes_port` 命令：

```rust
#[tauri::command]
async fn cmd_get_hermes_port(app: tauri::AppHandle) -> Result<Option<u16>, String> {
    let state: tauri::State<AppState> = app.state();
    let port = state.hermes_port.lock().await.clone();
    Ok(port)
}
```

### 2. `web/src/Splash.tsx`

强制跳转到 onboarding：

```tsx
useEffect(() => {
    // Always go to onboarding for now
    nav("/onboarding/welcome", { replace: true });
}, [nav]);
```

### 3. `web/src/onboarding/steps/Done.tsx`

修改 `openChat` 函数，使用 `cmd_get_hermes_port` 获取端口并跳转：

```tsx
async function openChat() {
    clearDraft();
    try {
      const port = await invoke<number>("cmd_get_hermes_port");
      if (port) {
        window.location.replace(`http://127.0.0.1:${port}/`);
      } else {
        console.error("Hermes port not available");
        window.location.replace("/");
      }
    } catch (e) {
      console.error(e);
      window.location.replace("/");
    }
}
```
